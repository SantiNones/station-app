# Station (MVP)

## Objective
Validate two hypotheses:
- Users want to reserve seats in cafés for work
- Cafés want to offer/manage reservations

## MVP Scope (strict)
User side:
- Café list (card grid)
- Café detail page
- Time slot selection
- Booking CTA
- Confirmation screen

Café side (simple):
- Define available slots (hardcoded/basic UI)
- View reservations

Out of scope:
- Payments
- Authentication
- Complex backend logic

## Validation requirements
Track:
- Reserve CTA clicks
- Completed bookings
- Drop-offs
